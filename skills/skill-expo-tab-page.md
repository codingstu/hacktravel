# Skill: expo-tab-page

关联文档：
- [`implementation-blueprint.md`](plans/implementation-blueprint.md)
- [`工程协作与Agent-Skills规范.md`](docs/2026-03-09/工程协作与Agent-Skills规范.md)

## 1. 适用场景

用于新增或调整 Expo 三 Tab 页面，包括路由文件、页面状态、默认内容、空态、加载态与失败态。

## 2. 前置条件

- 已明确所属 Tab 与页面目标
- 已定义页面输入输出
- 已确认是否依赖后端接口或本地 mock 数据

## 3. 操作步骤

1. 在路由目录新增或调整页面文件
2. 拆分页面与复用组件
3. 实现空态、加载态、失败态
4. 接入默认预置内容，避免首屏空白
5. 校验原生端与 Web 端表现一致性
6. 更新 `docs/YYYY-MM-DD/功能点.md`
7. 执行 Git 提交并 push

## 4. 验收标准

- 页面符合单一职责
- 三种状态显式可见
- 首屏存在默认内容
- 交互路径可完成闭环

## 5. 失败回退

- 退回上一版页面组件
- 暂时切换为 mock 数据渲染
- 关闭新入口但保留底层组件

## 6. 产出物

- 页面代码
- 复用组件
- 功能文档
- `feat(app): ...` 提交记录
