# Spec / 产品规格 — HackTravel

## 问题描述

旅行规划流程碎片化严重：用户需要在多个平台间跳转搜索、比价、拼凑行程，花费大量时间且容易遗漏信息。缺乏一站式、AI 驱动的行程规划工具。

## 目标用户

- **主要用户：** 自由行旅行者（18–35 岁），希望快速生成高质量行程。
- **关键特征：** 偏好极限体验（"极限爆改"）、习惯参考他人路线（"抄作业"）、关注价格波动（"盯盘"）。

## 核心流程

### 1. 行程生成（极限爆改）
- **触发：** 用户输入目的地、时长、预算、偏好标签
- **步骤：** 提交请求 → AI 模型生成行程 → Pydantic 校验 → 返回时间轴
- **预期结果：** 展示完整的分段行程，含交通、餐饮、景点、休息

### 2. 社区浏览（抄作业）
- **触发：** 用户进入抄作业 Tab
- **步骤：** 浏览热门路线 → 收藏 → 一键复制为自己的行程
- **预期结果：** 从社区路线中获取灵感，快速复用

### 3. 价格监控（盯盘）
- **触发：** 用户关注某条路线或目的地的价格
- **步骤：** 添加到监控列表 → 系统定期检查 → 价格变化通知
- **预期结果：** 用户在最佳时机出手

### 4. 一键导航
- **触发：** 用户在行程详情页点击导入
- **步骤：** 提取站点 → 生成 Google Maps 深链 → 跳转导航
- **预期结果：** 无缝从行程进入实际导航

## 非目标

- 不做机票/酒店预订闭环
- 不做社交聊天功能
- 不做复杂的用户权限体系
- 不做多语言 i18n（首期只做中文）

## 约束条件

- 匿名优先可用，登录入口默认关闭
- 主 AI 模型为 Codex 5.4，备路由为硅基流动、NVIDIA NIM
- 主备全失败时回退缓存路线
- 行程字段经 Pydantic 严格校验与归一化
- 错误码前缀 `HKT`

## 验收标准

- 行程生成端到端可跑通（前端提交 → 后端生成 → 前端展示）
- AI 超时 30s 内有缓存兜底
- 三个 Tab 首屏非空白，有预置内容
- Expo 可在 iOS/Android/Web 三端运行
