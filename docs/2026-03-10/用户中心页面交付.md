# 用户中心页面交付

## 目标

新增第四个 Tab「Profile / 我的」用户中心页面，与现有系统 Stitch 设计风格保持一致，对齐原型图实现以下区块：

1. 顶部标题栏（My Profile + 设置/退出图标）
2. 用户头像卡片（头像、姓名、标签、Edit Profile + 分享按钮）
3. 统计数据栏（TRIPS / SAVED / REVIEWS）
4. Active Price Alerts 价格提醒区块
5. Saved Itineraries 已保存行程卡片列表
6. Personal Preferences 个人偏好设置（Dark Mode / Language / Currency）

## 变更范围

| 文件 | 变更类型 | 说明 |
|------|---------|------|
| `frontend/app/(tabs)/profile.tsx` | 新增 | Profile 用户中心页面，含完整 UI 与 Mock 数据 |
| `frontend/app/(tabs)/_layout.tsx` | 修改 | 新增第四个 Tab（Profile），使用 person 图标 |
| `frontend/services/i18n.ts` | 修改 | 新增 31 个 profile.* 翻译键（zh/en 双语） |

## 关键实现

### 页面结构（profile.tsx）

- **顶部标题栏**：左侧设置齿轮图标、中间标题、右侧退出图标
- **用户头像卡片**：圆角 xl 白色卡片，包含 100px 圆形头像（带相机编辑图标）、用户名、认证标签、Edit Profile 主色按钮 + 分享按钮
- **统计数据栏**：三列等分布局，primary 色数字 + 大写标签
- **价格提醒卡片**：圆角 xl 卡片，左侧通知图标、中间路线信息、右侧降价金额（绿色）
- **已保存行程卡片**：圆角 xl 卡片，顶部目的地封面图（复用 `getDestinationImage`）、右上角书签图标、底部标题与元信息
- **个人偏好设置**：三行设置项，Dark Mode 使用 Switch 组件，Language 和 Currency 使用 chevron 箭头导航

### 设计风格对齐

- 复用 `@/constants/Theme` 中的 `Colors`、`Spacing`、`FontSize`、`FontWeight`、`BorderRadius`、`Shadow`
- 复用 `@/services/images` 中的 `getDestinationImage` 获取行程封面图
- 复用 `@/services/i18n` 中的 `t()` 函数实现多语言
- 圆角 xl(24) 卡片、primary/10 背景色、极轻投影，与其他三个 Tab 完全一致

### 状态管理

- 显式实现空态、加载态、错误态三种状态
- 已移除 Mock 数据，全面对接后端真实 API
- 使用 `AsyncStorage` 存储生成的匿名 `device_id` 作为用户标识
- Dark Mode / Language / Currency 偏好设置已实现与后端的双向同步，并支持乐观更新
- 实现了下拉刷新功能（`RefreshControl`）
- 实现了删除已保存行程的交互与 API 对接

### 后端 API 实现

- **数据层**：基于 Redis 持久化，使用 Hash、ZSet 等数据结构存储用户资料、统计数据、偏好设置和已保存行程
- **服务层**：`ProfileService` 封装了所有 Redis 操作，处理了默认值初始化和数据类型转换；新增 `email` 字段用于提醒查询
- **路由层**：新增 `/v1/profile` 系列接口，包含 GET/PUT 资料、GET 统计、GET/PUT 偏好、GET/POST/DELETE 行程
- **模型层**：使用 Pydantic 定义了完整的请求和响应模型，包含严格的字段校验

### 关键修复（可交付）

- **Web 点击无响应修复**：Web 端禁用 `RefreshControl`，避免拦截所有触摸事件
- **编辑资料**：新增可编辑弹窗并保存到后端（姓名/签名/邮箱/到访国家）
- **分享**：Web 端复制链接到剪贴板；原生端使用系统分享
- **设置**：点击齿轮自动滚动到偏好设置区
- **退出登录**：清除 `device_id` 并重载 Profile 数据
- **提醒列表**：基于邮箱调用 `/v1/watchlist/alerts`，弹窗展示提醒列表
- **行程管理**：弹窗展示已保存行程，支持删除
- **偏好设置**：语言/货币/深色模式即时生效并持久化

### Tab 布局变更

- 底部 Tab 从 3 个扩展为 4 个：Plan / Guides / Radar / Profile
- Profile Tab 使用 `person` / `person-outline` 图标，与其他 Tab 风格一致
- 选中态使用 `activeIconWrap` 样式（primary/10 圆形背景）

## 风险与回滚

- **风险**：当前使用匿名 `device_id`，如果用户清除 App 数据会导致资料丢失；Dark Mode 切换目前仅更新了后端偏好和本地状态，全局主题切换机制（如 Context/Provider）需在后续迭代中完善。
- **回滚方式**：前端还原 `profile.tsx` 到 Mock 版本，后端移除 `profile` 相关的路由和模型即可。
- **后续计划**：接入正式的用户认证系统（如 OAuth），将 `device_id` 迁移或绑定到真实用户 ID；完善全局 Dark Mode 主题切换。
