# 社区交互修复与首页体验优化

## 1. 功能目标

- **社区 Tab 筛选**：修复"旅行攻略"页面四个类别 Tab（精选 / 热门 / 最多收藏 / 省钱）点击无反应的问题，实现真正的分类筛选与排序逻辑。
- **社区筛选标签**：将四个标签 Chip（穷游 / 吃货 / 徒步 / 拍照）从静态 View 改为可切换的 TouchableOpacity，并实现基于中文关键词的路线匹配过滤。
- **详情报错修复**：修复社区路线点击展开详情时的报错（ID 不匹配 + Place Detail 查询失败未做兜底）。
- **首页 Hero 缩小**：将首页"下一站去哪"Hero 区域从 220px 压缩至 100px，减少首屏无效空间。
- **城市自动补全**：在出发地和目的地输入框增加自动补全下拉（支持约 80 个城市的拼音/中文模糊匹配）。
- **图片覆盖扩展**：补充 14+ 个缺失目的地的 Unsplash 图片映射，新增 onError 兜底逻辑。
- **前端预置路线同步**：将前端 presets.ts 从 6 条路线扩展至 28 条，覆盖后端 preset_routes.py 中全部六大洲路线。

## 3. 关键实现

### 3.1 社区 Tab 分类筛选

**问题根因**：原有代码使用 `i === 1` 硬编码 active 状态，无状态管理，点击任何 Tab 都不会触发数据变化。

**修复方案**：
- 引入 `CategoryTab` 类型：`'verified' | 'hot' | 'bestSellers' | 'budget'`
- 添加 `selectedCategory` 状态（默认 `'hot'`）
- 创建 `displayRoutes` useMemo：
  - `hot`：按 copy_count 降序
  - `bestSellers`：按 copy_count 降序
  - `budget`：按 budget.amount 升序
  - `verified`：显示全部
- Tab 组件改为交互式 `TouchableOpacity`，active 状态由 `selectedCategory === tab.key` 动态决定

### 3.2 社区筛选标签（Filter Chips）

**问题根因**：原有 Chip 使用 `<View>` 组件，无点击事件绑定；active 状态使用 `i === 0` 硬编码。

**修复方案**：
- 引入 `FilterTag` 类型和 `FILTER_TAG_MATCH` 映射表
- 映射关系：
  - `budget` → `['穷游', '穷鬼', '省钱', '极限']`
  - `foodie` → `['吃货', '美食', '小吃', '夜市']`
  - `hiking` → `['暴走', '徒步', '漫游', '古城']`
  - `photo` → `['打卡', '拍照', '网红', '电影感']`
- 在 `displayRoutes` useMemo 中进行关键词匹配过滤
- 筛选结果为空时显示空态 UI（带重置按钮）

### 3.3 详情报错修复

**问题 1 — Place Detail 查询失败**：
- 原有 `handlePressPlace` 的 catch 块为空注释
- 修复：catch 中构造 fallback `PlaceDetailResponse`，包含 Google Maps URL

**问题 2 — 预置路线 ID 不匹配**：
- 前端 ID 格式为 `preset-okinawa-24h`
- 后端 ID 格式为 `community-冲绳-48h`（由 `_make_route_id()` 生成）
- 修复：`_fallbackToPresets` 中 fallback ID 改为 `community-${r.destination}-${r.total_hours}h`
- `toggleExpand` 中增加 `_findPresetDetail(id)` 辅助函数，同时匹配 community 风格 ID 和原始 ID

## 5. 风险与回滚

- **自动补全性能**：`ALL_CITIES` 约 80 条 + `filterCities` 使用 `includes` 匹配，数据量小无性能风险。如需回滚，删除 `ALL_CITIES` / `filterCities` / 下拉相关 JSX 和 styles 即可。
- **预置路线体积**：presets.ts 从 ~250 行增至 ~600 行，对 bundle size 影响约 10KB（gzip 后 ~2KB），可接受。
- **图片 CDN**：Unsplash 的 `source.unsplash.com` CDN 可能在部分地区访问慢，onError 兜底保证不会白屏。
- **筛选逻辑**：`FILTER_TAG_MATCH` 的关键词匹配依赖中文字符串，后续如 tag 变更需同步更新映射表。
