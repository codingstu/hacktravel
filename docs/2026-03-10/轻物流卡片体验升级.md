# 轻物流卡片体验升级

## 1. 功能目标
- 统一底层 Tab 结构与多 Tab 页面头部，将窗口视觉拉回「轻物流卡片」风格：下一代 Tab 底栏、Hero 区域、状态卡片和标签都要统一圆角、柔和渐变与微妙投影。
- 让「行程生成」「社区精选」「Watchlist 实时跟踪」三大主流程在内容和视觉上保持一致（渐变胶囊、Hero 指标、Live 状态、卡片边框），同时补齐 theme 常量与辅助 hook 的使用。
- 以新的设计系统 Tokens（颜色 / 阴影 / border radius）为支撑，把文档/提交模板里的「先有文档、再 commit、再 push」闭环落实到本次迭代。

## 2. 变更范围
- `frontend/app/(tabs)/_layout.tsx`：Tab 栏重新抬升、加入边框与阴影、Active Icon 适配、跨平台安全高度。
- `frontend/app/(tabs)/index.tsx`：行程生成页 Hero 头部升级为渐变背景、增加 metrics 卡片、修正标签与自定义组件。
- `frontend/app/(tabs)/community.tsx`：社区头部改为渐变背景、增加 pill 状态、强化卡片边框与标签状态。
- `frontend/app/(tabs)/watchlist.tsx`：Watchlist Hero 增加次级行、LIVE 徽章与输入/卡片边框统一。
- `frontend/constants/Theme.ts`：引入新的 v6 设计系统 Tokens（色彩、渐变、阴影、边距），扩充状态色与标签风格。
- `frontend/components/ExternalLink.tsx`：补齐 `Link` 的 `href` 类型并保证 WebBrowser 接口稳定。

## 3. 关键实现
### 3.1 Tab 栏重构
- `TabLayout` 使用 `Platform.select` 设置不同平台的高度与 padding，Tab Bar 本体改为圆角卡片，围绕 `Colors.tab.border` 添加边框和阴影，激活状态由透明 `BorderRadius.full` 环绕。
- `tabBarItemStyle` 统一圆角与左右 margin，让“浮动”效果在三 Tab 之间更明晰。
- Icon 状态背景加了 `borderWidth` 提升可触感，尺寸微调并保持栅格对齐。

### 3.2 生成页 Hero + 表单
- Hero 区块切换成 `LinearGradient`，在顶部插入品牌徽章/菜单按钮，并新增 `HeroMetric` 组件展示时长、预算、区域三条核心数据。
- 文案由“去哪儿穷游爆改”调整为“Plan Smart Travel Better”，副标题与按钮色调统一为 `Colors.textOnDark` 与 `Colors.accent`。
- 节点组合和标签去除 emoji，切换到全文字表达并使用 `Colors.tag`/`tagActive` 体系让选中状态对比更加柔和。
- 表单与地图按钮加上轻投影、边框，`TextInput` 采用 `surfaceElevated` 背景与 `Colors.border` 轮廓。

### 3.3 社区精选 Hero + 卡片
- 社区头部改用 `LinearGradient`、顶部插入状态 pill（Community Picks + live indicator），Live 指示器在 `Colors.status` 中定义背景与文案色。
- 卡片轮廓从纯面板改为细线边框，标签、徽章、状态 chip 都用统一的 `Colors.tag`/`tagActive` 体系，提升视觉一致性。

### 3.4 Watchlist Hero 与警报卡
- Hero 区块增加 top row（包含 Flight Radar pill + LIVE badge）与副标题“实时盯盘 · 底价捕手”。
- 所有输入框、Tag、Alert 卡片统一加上边框并使用 `surfaceElevated`/`border` 组合，快速标签沿用 tag style 以确保一致性。
- 告警卡片左侧 border、状态色使用 `Colors.status.liveBg/liveText`，`alertStatusActive` 复用统一状态色。

### 3.5 设计系统升级
- `Colors` 从 v5 蓝白升级到 v6 轻物流卡片方案：柔和信任蓝、统一标签/状态/Tab/gradient 体系。
- `BorderRadius`、`Shadow` 数值微调，确保 Tab、card Hero 与浮层边缘视觉一致，新的 `Shadow.colored` 抽象函数保持可复用性。
- `Colors.status` 命名让 LIVE / 进度相关色更容易在组件中复用。

### 3.6 ExternalLink 强化
- `ExternalLink` 新增 `LinkHref` 类型，保证在 WebBrowser 模式下传入的 href 总能被转为字符串，避免 Web 环境下的联合类型擦除问题。

## 4. 配置项变更
- 无新增运行时配置。

## 5. 风险与回滚
- 主题色/Shadow 数值改动可能在其他复用页面引起对比或投影不一致，若出现可在 `Theme.ts` 中回退为 v5 方案并重新生成对应 `Colors.gradient`。
- Hero 以及 Tab Bar 的边框与渐变新增可能影响不同屏幕上的布局，回滚手段是恢复 `GenerateScreen`、`CommunityScreen`、`WatchlistScreen` 旧版 hero 样式并删掉新增的 LinearGradient 内容。

## 6. 验收结果
- [x] 三个 tab 页的 Hero 区、Live 标记、标签/卡片边框等视觉效果使用统一圆角+渐变+状态色。
- [x] Tab 栏呈现浮动卡片样式，激活图标背景有明确边框。
- [x] 设计系统 Tokens 扩充并在 app/Constants 中完成更新。
- [x] ExternalLink 类型更新避免 WebBrowser 类型擦除，同时插入 LinearGradient 的 tab 页面都能正常渲染。
