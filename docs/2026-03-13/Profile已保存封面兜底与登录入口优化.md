# Profile 已保存封面兜底与登录入口优化（2026-03-13）

## 背景 / 问题
- AI 规划保存行程后，Profile 的 Saved Itineraries 卡片区域出现“无图”（外网图片源不可用时更明显）。
- 未登录状态下，Profile 顶部右上角仍展示“退出”入口，语义不符合当前默认未登录。
- 删除收藏时出现二次操作体验（点击删除按钮容易触发卡片点击/打开详情）。
- 左上角设置按钮信息密度低（鸡肋）。

## 目标
- Saved Itineraries **始终有图**：即使远程封面加载失败，也要有本地兜底图。
- **未登录显示登录/注册入口**；仅在已登录时显示退出。
- 删除收藏 **一次点击完成**，避免触发卡片点击导致的“二次操作”。
- 精简顶部栏，移除低价值入口。

## 方案
### 1) 封面图兜底
- 渲染优先级：
  1. `itinerary.cover_image`（保存时写入的封面 URL）
  2. `getDestinationImage(destination)`（目的地远程兜底）
  3. 本地图片 `assets/images/splash-icon.png`（当远程加载失败时兜底）
- 当 `Image` 触发 `onError` 时，按 `itinerary_id` 记录失败并切换到本地兜底图。

### 2) 登录/退出入口
- Profile 顶部右上角：
  - `isLoggedIn=false`：显示“登录”图标并打开 `AuthScreen`。
  - `isLoggedIn=true`：显示“退出”图标与退出确认弹窗。
- 退出登录不再清除 `device_id`，避免匿名收藏随退出一起“丢失”。

### 3) 删除收藏一次完成
- 删除按钮 `onPress` 增加 `stopPropagation`（在 Web/可用平台），避免同时触发卡片 `onPress` 导致打开详情。

### 4) 顶部栏简化
- 移除左上角设置按钮（保持标题居中，用占位 View 保持布局稳定）。

## 影响范围
- 仅前端 Profile 页交互与展示层调整；后端接口与数据契约无变更。

## 验证方式
- 前端质量门控：
  - `cd frontend && npx tsc --noEmit`
  - `cd frontend && npx jest --runInBand`
- 手动回归建议：
  - 保存行程 → 切到 Profile → Saved Itineraries 卡片应有封面（远程失败也有本地兜底）。
  - 未登录：右上角为登录入口；登录后：右上角变为退出。
  - 点击卡片右上角书签删除：不应再打开详情弹窗。

## 已知限制
- 若用户网络环境对多数图片源不可达，本次兜底会统一回退到本地默认图；不保证每个目的地都有差异化封面。
