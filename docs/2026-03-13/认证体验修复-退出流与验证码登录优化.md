# 认证体验修复：退出流与验证码登录优化

日期：2026-03-13

## 背景

用户反馈以下问题影响登录体验与可用性：

1. 已通过验证码注册并登录后，退出登录仍停留在 Profile 且显示默认 mock 数据。
2. 登录页“忘记密码”没有直接可用的验证码登录路径。
3. Web 端输入框存在浏览器默认蓝色焦点框，且可点击区域偏窄，交互体验差。

## 本次改动

### 1) Profile 退出登录状态流修复

文件：`frontend/app/(tabs)/profile.tsx`

- `loadData` 增加登录态守卫：未登录时不再拉取 Profile 数据，不再落入 mock 默认值。
- `useFocusEffect` 仅在已登录时刷新数据；未登录时清理 Profile 相关状态。
- 新增未登录视图（登录提示 + 登录/注册按钮），确保退出后回到登录态体验。
- `confirmLogout` 不再调用 `loadData`，改为：
  - 清理本地资料/收藏/详情状态
  - 关闭退出确认
  - 直接打开 Auth 弹窗
- Auth 弹窗新增 `onSuccess` 回调，登录成功后立即刷新 Profile 数据。

### 2) 登录页验证码登录入口优化

文件：`frontend/components/AuthScreen.tsx`

- 登录模式新增邮箱登录方式 Tab：
  - `密码登录`
  - `验证码登录`
- “Forgot Password?” 现在可直接切换到验证码登录路径（无密码可用）。
- 验证码模式可一键切回密码模式。
- 输入框样式增加更稳定的高度策略：`minHeight` + 输入区 `height: 100%`，提升可点击区域。

### 3) Web 输入框全局交互修复

文件：`frontend/app/+html.tsx`

- 新增输入控件全局 reset：
  - 去掉浏览器默认 `outline` / `box-shadow`（蓝框）
  - 统一 `appearance` / `font` / `line-height`
  - `input:focus` 明确禁用默认焦点框

## 验证

执行：

```bash
cd frontend && npx tsc --noEmit && npx jest --runInBand
```

结果：

- TypeScript 类型检查通过
- Jest 测试通过（2 suites, 26 tests）

## 风险与后续

- 当前 Web 输入框 reset 采用全局策略，后续如引入第三方 Web 原生表单控件，需回归检查焦点样式是否符合预期。
- 建议后续补一条前端 E2E：`登录 -> 退出 -> 回到登录态 -> 验证码登录`。
