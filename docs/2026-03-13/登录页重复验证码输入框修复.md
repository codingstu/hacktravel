# 登录页重复验证码输入框修复

日期：2026-03-13

## 问题

在登录页切换到“验证码登录”后，界面出现两个验证码输入框：

- 上方 `Verification Code` 输入框
- 下方 `Email Code + Send Code` 输入框

这会造成输入重复、心智混乱，且与单验证码登录流程不一致。

## 根因

`AuthScreen` 中登录模式下的渲染条件存在重叠：

- 一处在 `emailLoginMode === 'code'` 时单独渲染验证码输入。
- 另一处在 `(mode === 'register' || emailLoginMode === 'code')` 时再次渲染邮箱验证码区块。

导致登录态下同一字段 `emailCode` 被两个输入框同时承载。

## 修复

文件：`frontend/components/AuthScreen.tsx`

- 登录态仅保留一个验证码输入区（含发送验证码按钮）。
- 密码区块仅在 `register` 或 `emailLoginMode === 'password'` 时渲染。
- 验证码区块在 `register` 或 `emailLoginMode === 'code'` 时渲染。
- 保留 `Forgot Password? Use Code` 与 `Use Password` 的模式切换入口，确保逻辑闭环。
