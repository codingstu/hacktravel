# 生产部署落地（VPS + Docker Compose + Caddy + Vercel KV）

## 1. 功能目标

- 新增生产级 Docker Compose 配置，将后端（FastAPI + Uvicorn）通过 Caddy 反代对外暴露 `https://api.hacktravel.app`。
- 前端 Web 已独立部署至 Vercel（静态导出）；本次仅覆盖后端 VPS 部署。
- 提供 `.env.prod.example` 模板，规范生产环境变量管理，避免真实密钥入库。

## 2. 变更范围

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `docker-compose.prod.yml` | 新增 | 生产 Compose：仅 `api` + `caddy`，`api` 仅 expose 8000 到 Docker 网络 |
| `Caddyfile` | 新增 | 反代 `api.hacktravel.app → api:8000`，gzip、安全头、自动 HTTPS |
| `backend/.env.prod.example` | 新增 | 生产环境变量模板（不含真实密钥） |
| `plans/deployment-plan.md` | 新增 | 完整部署通用清单文档 |

## 3. 关键实现

### 3.1 拓扑说明

```
User ──HTTPS──> Caddy Container (443/80)
                    └──HTTP (Docker network)──> api Container (:8000)
                                                    ├── Vercel KV (Upstash Redis, TLS)
                                                    ├── Supabase Postgres (预留)
                                                    └── LLM Gateways (HTTPS)
```

- `api` 服务仅 `expose: 8000`，不映射到宿主机，只对同 Docker 网络的 Caddy 可见。
- `caddy` 映射 `80:80` 和 `443:443`，负责 Let's Encrypt 自动签发证书（HTTP-01 校验）。
- Redis（Vercel KV/Upstash）和 Postgres（Supabase）全部托管，VPS 仅运行应用容器。

### 3.2 环境变量强依赖

生产必填项（见 `backend/.env.prod.example`）：
- `APP_ENV=production` / `DEBUG=false`
- `REDIS_URL=rediss://...`（Upstash Redis，TLS 连接串）
- `LLM_PRIMARY_API_KEY` 等 LLM Keys

可选/预留：
- `DATABASE_URL`（Supabase；当前代码未实际使用，建议先填占位）

### 3.3 CORS（后续需收紧）

当前后端 `allow_origins=["*"]`（见 `backend/app/main.py`）。
生产建议收紧为 `["https://hacktravel.app", "https://www.hacktravel.app"]`，避免任意站点调用 API。

## 4. 配置变更

- `docker-compose.prod.yml`：`api.expose=["8000"]`，`caddy.ports=["80:80","443:443"]`
- `Caddyfile`：`reverse_proxy api:8000`，域名 `api.hacktravel.app`

## 5. 验收结果

预期验收命令（在 VPS 上执行，DNS 已指向 VPS）：

```bash
curl -s https://api.hacktravel.app/health
# 预期：{"status":"ok",...}
```

## 6. 风险与回滚

| 风险 | 处置 |
|------|------|
| `.env` 含真实 LLM Key | 已在 deployment-plan 中标注必须移除并轮换；`.env.prod` 不入库 |
| DNS 未生效导致证书申请失败 | 先确认 `api.hacktravel.app` A 记录已指向 VPS，80/443 端口已放通 |
| Redis 连接失败 | 检查 `REDIS_URL` 是否使用 `rediss://`（TLS），Upstash 控制台确认连接串 |
| 回滚 | `git checkout <上一个 commit>` + `docker compose -f docker-compose.prod.yml up -d --build` |
