## 1. 功能目标

交付 M1 后端最小链路，对应蓝图 Day 1-3 排期，包括：
- 行程生成 API（POST /v1/itineraries/generate）
- Pydantic 请求/响应严格校验
- LLM 网关多供应商自动容灾（Codex 5.4 → 硅基流动 → NVIDIA NIM）
- Redis 缓存层 + query_hash 归一化 + 幂等键保护
- Google Maps URL 拼接（含分段策略）
- 统一错误码（HKT 前缀）
- 匿名限流（滑动窗口 10 次/小时）

## 3. 关键实现

### 3.1 行程生成 API

- 对齐蓝图 Section 3.1/3.2 契约
- 请求体含 origin、destination、total_hours、budget、tags、locale、timezone、idempotency_key
- 响应含 itinerary_id、title、summary、legs 时间轴、map deeplink、source 溯源、policy

### 3.2 LLM 网关容灾

- 三级供应商链：codex54 → siliconflow → nvidia
- 触发条件：超时（主 15s / 备 20s）、5xx、连接异常
- temperature = 0.3
- Prompt 含系统角色（资深旅行规划师）+ JSON 骨架强约束 + stop sequence
- JSON 提取三级策略：直接解析 → 正则 code block → 花括号定位

### 3.3 缓存与幂等

- query_hash 归一化：destination + hours + budget(50 取整) + currency + tags 排序
- 缓存 TTL：热门 7 天 / 冷门 3 天
- 幂等键 10 分钟窗口
- 限流：匿名 10 次/小时，Redis INCR + EXPIRE

### 3.4 Google Maps URL

- 完整格式：`https://www.google.com/maps/dir/?api=1&origin=...&destination=...&waypoints=...|...&travelmode=transit`
- waypoints ≤10 单 URL，超限自动分段
- 优先经纬度，回退地名编码

### 3.5 错误码

| 错误码 | HTTP | 场景 |
|--------|------|------|
| HKT_400_INVALID_INPUT | 400 | 参数不合法 |
| HKT_409_DUPLICATE_REQUEST | 409 | 幂等重复 |
| HKT_422_SCHEMA_VALIDATION_FAILED | 422 | 模型输出校验失败 |
| HKT_429_RATE_LIMITED | 429 | 超限 + Retry-After |
| HKT_503_MODEL_UNAVAILABLE | 503 | 模型不可用 |
| HKT_504_MODEL_TIMEOUT | 504 | 模型超时 |
| HKT_599_FALLBACK_CACHE_MISS | 599 | 全失败且无缓存 |

- PostgreSQL 数据表尚未创建（Alembic 迁移将在下一功能点补充）

- 删除 `backend/` 目录与 `docker-compose.yml` 即可完全回滚
