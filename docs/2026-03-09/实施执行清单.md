# HackTravel 实施执行清单

关联文档：
- 蓝图：[`implementation-blueprint.md`](plans/implementation-blueprint.md)
- 协作规范：[`工程协作与Agent-Skills规范.md`](docs/2026-03-09/工程协作与Agent-Skills规范.md)

## 1. 目标

把 Agent、Skills、Prompt、代码规范、文档更新、Git 提交流程变成可直接执行的日常 SOP。

## 2. 功能完成后的强制流程

每完成一个功能点，必须按以下顺序执行：

1. 完成功能开发与自测
2. 新增文档：`docs/YYYY-MM-DD/功能点.md`
3. 文档写入变更文件、验收结果、回滚方式
4. 提交代码：`feat(scope): 摘要`
5. push 到 `main`
6. 在功能文档中补充 commit hash

## 3. 功能文档命名与内容规范

### 3.1 命名规范

- 路径：`docs/YYYY-MM-DD/功能点.md`
- 示例：`docs/2026-03-09/行程生成接口.md`

### 3.2 必填章节

- 功能目标
- 变更范围
- 关键实现
- 配置项变更
- 验收结果
- 风险与回滚
- Git 记录

## 4. 提交信息规范

固定格式：`feat(scope): 摘要`

scope 建议：
- `api`
- `app`
- `llm`
- `maps`
- `community`
- `docs`
- `infra`

示例：
- `feat(api): 新增行程生成接口与参数校验`
- `feat(llm): 增加主备模型超时切换`
- `feat(docs): 新增行程生成接口功能文档`

## 5. 日常执行检查单

- [ ] 功能点边界明确
- [ ] 代码已完成并通过本地验证
- [ ] 文档已新增到 `docs/YYYY-MM-DD/功能点.md`
- [ ] 文档已登记配置项与回滚策略
- [ ] commit message 符合 `feat(scope): 摘要`
- [ ] 已 push 到 `main`
- [ ] 文档已记录 commit hash

## 6. 禁止项

- 一个 commit 混入多个无关功能点
- 只改代码不更新 docs
- 提交信息含糊不清
- 未完成验证直接 push

## 7. 与蓝图一致性

所有实现必须满足 [`implementation-blueprint.md`](plans/implementation-blueprint.md) 约束：

- 匿名优先
- 登录入口默认关闭
- 主模型 Codex 5.4
- 备路由硅基流动与 NVIDIA NIM
- 超时与 5xx 自动切换
- 失败回缓存
- 三 Tab 架构与导图能力
