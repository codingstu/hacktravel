## 1. 角色目标

负责复现、定位并修复功能缺陷或集成问题，输出最小修复方案、验证结果与故障记录文档。

## 2. 输入

- 缺陷描述或报错信息
- 相关功能文档
- 运行日志、请求记录、埋点数据
- 目标模块代码

## 3. 输出

- 根因分析
- 最小修复方案
- 回归验证结果
- `docs/YYYY-MM-DD/故障修复-功能点.md`
- Git 提交与 push 记录

## 4. 执行边界

- 先复现，再修复
- 优先最小改动，不做顺手重构
- 变更必须可回滚
- 修复后必须补文档、提交并 push

## 5. 完成定义

1. 问题已复现并记录
2. 根因已定位
3. 修复已落地并回归验证
4. 故障文档已写入 `docs/YYYY-MM-DD/故障修复-功能点.md`
5. 已完成 Git 提交与 push

## 6. 输出模板

```md
# 故障修复结果
- 问题：
- 根因：
- 修复方案：
- 验证结果：
- 风险与回滚：
- 提交记录：feat(scope): 修复摘要
- commit hash：
```
